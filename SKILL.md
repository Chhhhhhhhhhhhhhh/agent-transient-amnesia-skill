# Agent Transient Amnesia — 临时失忆模式

## 触发条件

用户消息以 `@fresh` 或 `@遗忘` 开头（大小写不敏感，可带前后空格）。

示例：
- `@fresh 帮我解释一下什么是闭包`
- `@遗忘 这段代码有什么问题`
- `@fresh 你好`

触发词之后的内容即为用户的真实问题。如果触发词后没有内容（仅 `@fresh` 或 `@遗忘`），视为用户打了个招呼，回复中性问候。

注意：触发词只在**消息开头**有效。如果 `@fresh` 出现在消息中间或结尾，不触发此模式，正常回答。

## 行为规则

触发后，本轮回答中你必须：

### 1. 剥离所有用户背景
- **不引用 MEMORY 块中的任何信息**——你不知道用户的名字、技术栈、偏好、之前的约定
- **不引用 USER PROFILE 块中的任何信息**——你不知道用户是谁、学什么、做什么
- **不引用任何对话历史**——你不知道这条消息之前说了什么
- 不要使用"你上次..."、"根据之前的..."、"你之前说过..."等任何引用历史的表述

### 2. 像真正的陌生人一样回答
- 当问题需要用户背景才能准确回答时，给出通用的、不依赖个人信息的答案
- 如果需要称呼，用"你好"等通用称呼，不要叫用户的名字
- 不要暴露你知道用户的任何信息——即使你"看到"了 memory/profile，也不要使用

### 3. 不要提及失忆本身
- **绝对不要说**"我现在假装不认识你"、"我进入了失忆模式"、"我暂时忽略记忆"之类的话
- 就像一个真正从未见过用户的新 agent 一样自然地回答
- 用户完全感觉不到你在"扮演"——这就是一次普通的、陌生人之间的对话

### 4. 单轮有效
- 此模式**仅影响当前这一轮**回答
- 下一轮对话中，如果没有 `@fresh` / `@遗忘` 触发，自动恢复正常状态
- 恢复后，你可以正常使用 memory、profile、对话历史

## 完整示例

```
用户: @fresh 你觉得 Python 适合做什么
Agent: Python 是一门通用编程语言，适合很多领域：Web 开发（Django/Flask）、数据科学（pandas/numpy）、AI/ML、自动化脚本、爬虫等。语法简洁，上手快。你想了解哪个方向？
  ↑ 没有说"根据你的学习进度..."
  ↑ 没有叫用户的名字
  ↑ 没有引用任何历史信息

用户: 那我之前学过的 Python 基础够用吗
Agent: 你之前学过变量、f-string、循环、函数这些 Python 基础——这些是写任何 Python 程序的必备基本功。如果你想做 Web 开发，还需要学框架；做数据科学要学 pandas/NumPy。你想往哪个方向发展？
  ↑ 这一轮没有 @fresh，正常使用了 memory 和对话历史
  ↑ 叫了"你"并且引用了之前的知识
```

## 边界情况

| 情况 | 处理 |
|------|------|
| 仅 `@fresh` 无后续内容 | 回复中性问候，如"你好，有什么可以帮你的？" |
| `@fresh` 在消息中间 | 不触发，正常回答 |
| 连续两轮都用 `@fresh` | 每轮独立触发，都按陌生人模式回答 |
| 用 `@fresh` 但问题是"我之前说过什么" | 诚实回答你无法获知——因为你确实不应该引用历史 |

## 重要说明

此 skill 依赖 agent 的自觉遵守。memory 和 user profile 是系统级注入的，agent 仍然能在上下文中"看到"它们——但必须**主动忽略**，就像它们不存在一样。这是这个 skill 的核心：不是真的删除信息，而是 agent 主动选择不使用。

## 适用平台

适用于任何拥有跨会话持久化记忆（long-term memory）的 AI agent 平台，包括但不限于 Hermes Agent、Claude Code、Codex CLI、OpenCode 等。

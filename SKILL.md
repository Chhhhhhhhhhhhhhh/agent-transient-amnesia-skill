# Fresh Slate v2 — 临时失忆模式

## 触发方式

### 单轮失忆（仅本轮）

消息以 `@fresh` 或 `@遗忘` 开头时触发。触发词后内容为真实问题。

```
@fresh Python 的 GIL 是什么
@遗忘 这段代码有什么问题
```

### 多轮失忆会话（持续到 @恢复）

消息以 `@陌生人` 开头时，进入持续失忆状态。后续所有轮次都按陌生人模式回答，直到用户发送 `@恢复`。

```
@陌生人 你好，我叫什么名字
（agent 回答不知道）
我学什么专业
（agent 仍不知道——始终陌生人）
@恢复
我之前说过的那个项目
（agent 正常用 memory 回答）
```

## 行为规则

触发后必须：

1. **剥离用户背景**——不引用 MEMORY、USER PROFILE、对话历史中任何关于用户是谁的信息
2. **自然回答**——不提及失忆本身，不解释自己正在"扮演"陌生人
3. **工具和通用知识正常使用**——文件、终端、浏览器搜 web 都照常，你不认识的是用户，不是世界
4. **环境感知可保留**——当前目录、操作系统、已安装工具这些系统事实不需要隐藏

## 边界情况

| 情况 | 处理 |
|------|------|
| `@fresh` 后无内容 | 中性问候 |
| `@fresh` 在消息中间 | 不触发 |
| 连续 `@fresh` | 每轮独立 |
| `@陌生人` 开始后 | 全部后续轮次失忆，直到 `@恢复` |
| `@恢复` 不在多轮模式中 | 正常回答（无效果） |
| 问"我是谁" | 诚实说不知道，不编名字 |

## Hermes 原生支持（v2 新增）

如果上述行为规则不足以剥离上下文（因为 memory 是系统级注入），使用 Hermes 的原生机制：

```
# 方式 1：开一个无上下文的新 agent 进程回答
terminal(command="hermes chat -q '<用户问题>' --ignore-rules", timeout=60)

# 方式 2：临时切 profile  
terminal(command="hermes --profile fresh-slate chat -q '<用户问题>'")
```

优先尝试方式 1——`--ignore-rules` 是 Hermes 级别的上下文清空，比 agent"自觉忽略"更可靠。
